# 纪律与留档(最高优先级,先读本文件再读盘)

## 一、管辖权与寿命

1. **一问一盘**:一张盘只回答一个问题。同一问题不重复起盘(越问越浑);想再问=回读旧盘做验证回填。
2. **管辖内可深挖**:同一件事的细节(钱/对方/过程/时间/物件描述)都属管辖内,用户要"再挖"就按 reading_rules 逐层翻。
3. **管辖外必须拒挖**:另一件事(哪怕相关)另起新盘或回本命盘。向用户明说这条规矩——这是保准度的代价。
4. **盘的寿命**:到"事情落定"为止。落定后标注【已收官】,此盘归档,只读不再解。
5. **长线问题不抢答**:人生方向/婚姻大势/事业格局属于本命盘的管辖,建议改用本命盘分析(上游仓库提供 vedic-core 等本命 skill)。prashna 是狙击枪,不是望远镜。

## 二、诚实条款(违反=废盘)

1. **先写后证**:一切可验证的预测写在【预注册】区,时间戳在结果之前。事后不改原文;对照结果只能补记在【现实对照】区。
2. **敢说不准**:信号弱/矛盾/月亮无力受克 → 明写"此盘含糊,不足以下判"。可选:建议改日再问或回本命盘。**准的前提是敢说不准。**
3. **来源分层**:每条结论标注来源——「prashna 盘面」/「本命盘/过运」/「用户透露的信息」。三者禁止混写(防事后诸葛污染验证)。
4. **词表≠遥视**:星座描述类输出必须标「象征层,待验证」。
5. **命中率由台账说话**:不拿单张盘的命中吹整体准度;向用户呈现的长期可信度=prashna_log.md 的累计统计。
6. **边界声明**:每份报告结尾写明——盘给方向/形状/窗口;现实尽调(验房/合同/就医/尽职调查)照常做;重大决定归用户。

## 三、输出模板(qa_prashna_<主题>.md)

```markdown

> 起盘:<YYYY-MM-DD HH:MM TZ>,<城市>(提问此刻,问主所在地)
> 引擎:True Chitra · vedic-calculator

## 一、盘面
<行星表:星座/宫/度数/逆行/星宿/尊贵 + 关键相位>

## 二、读盘推导
<归宫 → 连接判成败 → 动静 → 月亮剧本 → 时间 → 细节;每条亮出宫/星/规则>

## 三、判词
<方向 + 形状 + 窗口 + 唯一失败通道>

## 四、【预注册】可验证预测清单
<逐条编号,每条一个可判"中/不中"的具体预测,含词表类(标象征层)>

## 五、【现实对照】(留空待回填)
- (待事实回填:逐条对预注册编号记 ✅/❌/⚠️)

## 六、边界声明
<盘的管辖与寿命;现实动作照做;决定归问主>
```

## 四、台账(cases/<名字>/prashna_log.md)

每起一张盘追加一行;每次回填更新状态:

```markdown
| 日期 | 问题 | 判词(一句) | 预注册条数 | 验证 | 状态 |
|---|---|---|---|---|---|
| 2026-07-01 | 这次面试会拿到offer吗 | 倾向成;两周内有消息 | N条 | 待验 | 服务中 |
```

- 「验证」列最终写成 `✅x/❌y/⚠️z`(按预注册逐条计)。
- 台账是回答"prashna 到底准不准"的唯一合法依据。

## 五、伦理

- 健康类只判"这件事顺不顺",不诊断、不替代就医;生死/重病/创伤类话题措辞克制,不做恐吓式断言,不预测死亡。
- 涉第三方(对方/对手)只读盘面角色,不做人身论断。
- 用户情绪高涨时(狂喜/恐慌),提醒其"放大器"状态,判词照规则给,不迎合不吓唬。
