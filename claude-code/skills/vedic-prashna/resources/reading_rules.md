# 读盘规则(判成败 · 判动静 · 月亮剧本 · 时间 · 竞争)

## 一、判成败:连接为王

**核心问句:命主星(L1)/月亮,和事主星(karya 宫主)之间有没有"连接"?**

连接的形式(强→弱):
1. **同宫合相**(尤其 <8°) —— 最强
2. **互溶**(Parivartana,两星互换星座)
3. **互相相位**(Parashari 全相位:所有星对7宫位;火4/8、木5/9、土3/10 特殊相位)
4. 事主星落 1宫,或命主星落 karya 宫
5. 月亮与事主星的入相位(applying,见第三节)

判定:
- **有连接 + 事主星有力**(旺/庙/友方、不深度受克)→ **成**。
- **有连接 + 事主星弱**(陷/敌/被烧/战败)→ 成,但打折/费劲/结果缩水。
- **无连接** → 难成;除非 11宫(愿望宫)极强托底(吉星群聚/宫主有力照命),降级为"绕路可成"。
- 事主星或命主星落 6/8/12 且无补救 → 该事带对应色彩(6争斗/8危机反复/12流失暗耗),明说。

加分项(有则记):事主星在 11宫;吉星(木/金/水/盈月)落 1宫或 karya 宫;起盘日的曜日主 = 命主星或事主星("问得其时")。
减分项:凶星(土/火/Rahu/Ketu)以紧相位压 karya 宫或事主星(标出是哪种干扰:土=拖延负担、火=竞争冲突、Rahu=虚象放大、Ketu=心冷/中断);命主星深度燃烧;月亮无光(晦月前后)且受克 → 整盘可信度降级,启用铁规2"敢说不准"。

## 二、判动静:上升星座性质

| 上升性质 | 星座 | 对"变动/成行/搬移"类问题 | 对"维持/留下"类问题 |
|---|---|---|---|
| 活动(Chara) | 白羊/巨蟹/天秤/摩羯 | 动,快成行 | 难维持,状态会变 |
| 固定(Sthira) | 金牛/狮子/天蝎/水瓶 | 不动/拖 | 稳,维持 |
| 双元(Dvisvabhava) | 双子/处女/射手/双鱼 | **成,但经反复/商量/回环** | 变中有续 |

- 双元上升 + 命主星逆行 = "回环"信号叠加:不当场定,绕一圈再落(逆行=谨慎核验,**不等于慢走**,别让用户误读成拖延授权)。
- Prashna 上升恰好 = 问主本命上升,或复刻本命骨架(同轴 Rahu-Ketu 等)→ 标注"盘走到本命上,问得其所",可信度加档。

## 三、月亮剧本(事情如何展开)——本技法权重最高

**月亮接下来依次形成的入相位(applying aspects)= 事情接下来的分步剧本。**

算法(用引擎经度硬算,不许估):
1. 取月亮当前黄经 M(月速 ≈ 12-15°/天),取各行星黄经 P。
2. 对每颗行星,列出相位角 A ∈ {0, 60, 90, 120, 180}(prashna 用西式度数相位读展开,合相与对冲权重最高)。
3. 解 (P + A - M) mod 360 里**月亮尚未到达且在离开本星座前能到达**的最小正差 → 这些就是"接下来的接触",按先后排序。
4. 只取月亮**离开当前星座之前**的接触(出座即翻页)。

读法:
- 每个接触 = 一步剧情;**接触的星**说这一步是谁/什么(用它在本盘的宫职+自然属性),**相位性质**说这一步顺不顺(三合/六合=顺,刑/冲=要费劲面对,合相=直接遭遇)。
- 序列里**全是柔性相位** = 一路可走;中途有刑冲 = 那一步是坎(说清是哪类坎);**最后一个接触**往往是"结局的质感"。
- 月亮已刚离开某精确相位(separating,差<1°)= "已经发生/已定下的部分",可用来印证"这事其实已启动"。

## 四、时间:真实天象优先

给"何时"永远优先用**引擎逐日扫描真实事件**,而非象征折算:
- 命主星/事主星 **逆行转顺日**(station)= 拐点/绿灯;
- 命主星与事主星 **精确合相/相位完成日** = 回环闭合/落定窗;
- 关键星 **换座日** = 阶段切换。

扫描代码模式(swisseph,SIDM_TRUE_CITRA,逐日循环找 speed 变号 / 经度差过零 / 整除30过界)——本仓库 vedic-calculator venv 直接可用。
象征单位(活动=日、双元=周、固定=月)只在无法定位真实天象事件时作粗估,并标注"粗估"。

## 五、竞争盘(多人抢一个标的)

- 竞争是否存在:火星对 karya 宫的相位 / 6宫能量 → 有则明说"竞争真实"。
- 谁占优:命主星与事主星的连接紧密度+尊贵度 vs "场上其他人"(抽象为7宫方状态)。
- **唯一失败通道**:若盘整体判"成",找出全盘唯一能翻车的现实路径(常见:犹豫超窗被抢/资料不备/错过对方流程),**必须写出来**——这是 prashna 最有用的输出之一。
- 对方(7宫)有土星 = 对方流程慢、筛选严 → 对策土星化(资料齐、稳定人设);有 Rahu = 对方情况虚高/信息噪声大 → 核实再信。

## 六、事件时刻盘(辅助)

针对"已约定时刻的事件"(看房/面试/签约),可加算事件时刻盘,只读三样:
1. 事件上升与其主星(事件的第一印象/基调);
2. 上升点附近的行星(<3° 最强,如金星踩上升=卖相好);
3. 月亮的宫位与合相(现场的心绪与放大器,月合Rahu=提醒别被情绪放大器拍板)。
事件盘**不改主判**,只供现场提示。
